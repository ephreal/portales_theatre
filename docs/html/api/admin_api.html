theatre.routes.api.admin.create_play(play_data)¶

Creates a play with the info passed in.

The date must be passed in as a formatted string: %Y-%m-%d

The time must be passed in as a formatted string: %H:%M

Returns an error or the Play depending on whether or not the play creation was successful.

If no seat information was passed in, this will autogenerate an 8x12 grid of seats to go with the play.

Parameters:
play_data: JSON: {date, seats: {…}, description, name}
Returns:
Play, Dict

theatre.routes.api.admin.update_play(play_json)¶

Updates the play identified by play_id with the play_json. The play info should be JSON data and can be generated with the serialize_play() function from routes/route_functions.

This can return 3 different states.

Play: The updated play information False: The play was unable to be updated (ie: Errors were encountered) None: The play requested does not exist.

Parameters:
play_id: int
play_json: JSON {date, seats: {1: {seat JSON}, 2: …} }
Returns:
Play, False, None
theatre.routes.api.admin.update_seat(seat_id, seat_data)¶

Updates a single seat with the json passed in.

Currently the row/column values are not updated since the seat will most likely not be moved around after the seat is created.

Can return three types of values.

Seat: The updated version of the seat.

False: An error occurred in trying to update the seat.

None: The seat does not exist.

parameters:
seat_id: int
seat_data: JSON {price}
Returns:
Seat, False, None
